# WANI V2 프로젝트 지침

## 기술 스택
- CodeIgniter3, Bootstrap 5.3, jQuery 기반 구조를 유지하기
- 기존 코드 스타일, 라우팅, 헬퍼 사용 패턴을 우선 따르기

## 소스 참조 우선순위
- 이 저장소 코드를 먼저 확인하기
- GitHub 연결 소스와 MCP 서버 코드가 접근 가능하면 해당 내용을 우선 참조하기
- 추정으로 구현하지 말고 실제 코드 구조를 확인한 뒤 수정하기

## 화면/성능 원칙
- 페이지는 가능한 한 HTML 프레임 중심으로 구성하기
- 데이터 로딩과 동작은 AJAX 방식으로 처리하기
- 초기 렌더링과 네트워크 요청 수를 줄여 로딩 속도를 최적화하기

## 코드 제공 형식
- 수정 코드 설명은 함수 단위로 제공하기
- 각 코드 블록 최상단에 파일 경로와 코드 역할을 먼저 명시하기

## 설계 원칙
- MVC 패턴을 엄격히 지키기
- 유사 기능은 공통 함수/헬퍼/라이브러리로 통합하기
- 중복 함수 생성을 피하고 재사용 가능한 구조를 우선 설계하기
- `assets/js/common.js`의 공통 모듈을 우선 사용하고 동일 목적의 함수를 새로 만들지 않기
- 작업 중 `common.js`와 중복되는 로직이 발견되면 신규 중복 코드를 제거하고 공통 모듈 호출로 정리하기

## 응답/문구 정책
- 사용자 응답은 한국어로 작성하기
- 코드 description/메시지는 이모티콘 없이 평문으로 작성하기

## UI 상호작용 정책
- alert 대신 toast 메시지를 사용하기
- confirm 대신 모달을 사용하기

## 산출물 범위
- 가이드 문서와 테스트 코드는 사용자가 요청하기 전에는 제공하지 않기
